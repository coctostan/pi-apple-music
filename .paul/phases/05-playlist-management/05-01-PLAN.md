<objective>
## Goal
Extend the playlist tool to support adding tracks to existing playlists, and add a way to list existing playlists with their IDs so the LLM can select one to add tracks to.

## Purpose
Currently the extension can only create new playlists. Users want to add songs to playlists they already have. This requires: listing playlists with IDs, and an "add tracks" action on the playlist tool.

## Output
- `apple_music_create_playlist` tool extended with an `action` parameter: `"create"` (default) or `"add-tracks"`
- For `"add-tracks"`: accepts `playlistId` + `trackIds`, POSTs to `/v1/me/library/playlists/{id}/tracks`
- Library playlists action already returns playlist names — extend it to include playlist IDs so the LLM can reference them
</objective>

## API Reference
- Add tracks: `POST /v1/me/library/playlists/{id}/tracks`
  Request body: `{ data: [{ id: "track-id", type: "songs" | "library-songs" }] }`
  Response: 204 No Content
- Library playlists: `GET /v1/me/library/playlists` → response includes `id` field on each playlist
