<task type="auto">
  <name>Task 2: Implement all Spotify library functions with tests</name>
  <files>src/spotify/library.ts, test/spotify-library.test.ts</files>
  <action>
    **src/spotify/library.ts** — Create 9 exported functions mirroring src/library.ts patterns:

    All functions take `SpotifyClient` as first arg, return `Promise<string>` (markdown), use `spotifyCache`, and handle errors with try/catch returning error markdown.

    1. `fetchSpotifySavedTracks(client, limit=50)` — GET `/v1/me/tracks?limit={limit}`
       - Format: numbered list with `"name" by artist (album) duration (ID: id)`
       - Include track URI for playlist use

    2. `fetchSpotifySavedAlbums(client, limit=50)` — GET `/v1/me/albums?limit={limit}`
       - Format: `"name" by artist (N tracks, release_date)`

    3. `fetchSpotifyFollowedArtists(client, limit=50)` — GET `/v1/me/following?type=artist&limit={limit}`
       - Note: uses cursor-based pagination, different from offset-based
       - Format: `artist name [genre1, genre2]`

    4. `fetchSpotifyPlaylists(client, limit=50)` — GET `/v1/me/playlists?limit={limit}`
       - Format: `"name" (N tracks, ID: id) — description`

    5. `fetchSpotifyRecentlyPlayed(client)` — GET `/v1/me/player/recently-played?limit=20`
       - Format: `"name" by artist (played: relative time or timestamp)`

    6. `fetchSpotifyTopArtists(client, limit=20)` — GET `/v1/me/top/artists?limit={limit}&time_range=medium_term`
       - Format: ranked numbered list with `artist [genres]`

    7. `fetchSpotifyTopTracks(client, limit=20)` — GET `/v1/me/top/tracks?limit={limit}&time_range=medium_term`
       - Format: ranked `"name" by artist (album)`

    8. `fetchSpotifyGenreBreakdown(client, limit=50)` — GET `/v1/me/tracks?limit={limit}`, then for unique artist IDs batch GET `/v1/artists?ids={comma-separated}` (max 50 per request)
       - Aggregate genres from artist objects
       - Return markdown table: `| Genre | Artists | % |`

    9. `fetchSpotifyLibrarySummary(client)` — parallel fetch tracks, albums, playlists (limit=10 each)
       - Format: counts + sample listings (same pattern as Apple Music summary)

    **Helper:** `formatDuration(ms)` — reuse same format as Apple Music (M:SS)

    **test/spotify-library.test.ts** — Tests for all 9 functions:
    - Mock `SpotifyClient.get()` by creating a mock object with a `get` method (do NOT mock global fetch — the client is the abstraction boundary)
    - For each function test:
      - Success case: returns formatted markdown with expected fields
      - Empty case: returns "No X found" message
    - For `fetchSpotifyGenreBreakdown`: test the artist ID batching (>50 unique artists splits into multiple requests)
    - For `fetchSpotifyLibrarySummary`: test parallel fetch returns combined output
    - Target: ~20-25 tests covering all functions
  </action>
  <verify>npx tsc --noEmit && pnpm test 2>&1 | tail -10</verify>
