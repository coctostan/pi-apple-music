## AC-1: Catalog Search
```gherkin
Given a SpotifyClient with valid auth
When searchSpotifyCatalog(client, "bohemian rhapsody", 10) is called
Then it searches GET /v1/search with type=track and returns a markdown-formatted list of matching tracks with name, artist, album, and Spotify URI
```

## AC-2: Create Playlist
```gherkin
Given a SpotifyClient with valid auth and a Spotify user ID
When createSpotifyPlaylist(client, name, description, trackUris) is called
Then it creates a new playlist via POST /v1/me/playlists
And adds tracks via POST /v1/playlists/{id}/tracks
And returns a confirmation with playlist name, track count, and playlist URL
```

## AC-3: Add Tracks to Existing Playlist
```gherkin
Given a SpotifyClient with valid auth and a playlist ID
When addTracksToSpotifyPlaylist(client, playlistId, trackUris) is called
Then it adds tracks via POST /v1/playlists/{id}/tracks with track URIs
And returns a confirmation with playlist ID and track count
```

## AC-4: Duplicate Detection
```gherkin
Given a playlist with existing tracks
When creating a playlist or adding tracks
Then the function checks for duplicate URIs in the user's saved tracks (best-effort)
And reports duplicates in the response
```

## AC-5: Error Handling
```gherkin
Given an API error during search, create, or add-tracks
When the error occurs
Then the function returns a descriptive error message in markdown format without throwing
```

  <name>Task 2: Implement Spotify search, create playlist, and add tracks with tests</name>
  <files>src/spotify/playlist.ts, test/spotify-playlist.test.ts</files>
  <action>
    **src/spotify/playlist.ts** — 3 exported functions:

    1. `searchSpotifyCatalog(client, term, limit=10)`:
       - GET `/v1/search?q={encoded_term}&type=track&limit={limit}`
       - Note: Spotify Feb 2026 change reduced max limit to 10 for search
       - Format results as numbered list: `"name" by artist (album) — URI: spotify:track:id`
       - Include instruction text about using URIs with spotify_playlist tool
       - Cache results with spotifyCache

    2. `createSpotifyPlaylist(client, name, description, trackUris, existingTrackUris?)`:
       - Step 1: POST `/v1/me/playlists` with `{ name, description, public: false }`
       - Step 2: If trackUris.length > 0, POST `/v1/playlists/{id}/tracks` with `{ uris: trackUris }`
       - Spotify uses full URIs (`spotify:track:id`) not bare IDs
       - Best-effort dedup: if existingTrackUris provided, report overlap count
       - Return confirmation with playlist name, track count, URL from response

    3. `addTracksToSpotifyPlaylist(client, playlistId, trackUris)`:
       - POST `/v1/playlists/{playlistId}/tracks` with `{ uris: trackUris }`
       - Return confirmation with playlist ID and track count

    **test/spotify-playlist.test.ts** — Tests using mock SpotifyClient (same pattern as spotify-library tests):

    For searchSpotifyCatalog:
    - Returns formatted results with name, artist, album, URI
    - Handles empty results
    - Handles API error

    For createSpotifyPlaylist:
    - Creates playlist and adds tracks (verify two POST calls)
    - Reports duplicate count when existingTrackUris provided
    - Works without description
    - Handles empty track list
    - Handles API error

    For addTracksToSpotifyPlaylist:
    - Adds tracks and returns confirmation
    - Handles single track grammar
    - Handles API error
  </action>
