## Goal
Create the Spotify auth infrastructure (PKCE Authorization Code flow) and HTTP API client, mirroring the Apple Music client architecture.

## AC-1: Spotify Config Loading
```gherkin

When loadSpotifyConfig() is called
Then it returns a SpotifyConfig object with clientId, accessToken, refreshToken, and expiresAt
```

## AC-2: Spotify Config Validation
```gherkin
Given a SpotifyConfig object
When isSpotifyConfigured() is called
Then it returns true only if clientId and accessToken are present and non-empty
```

## AC-3: Token Refresh
```gherkin
Given an expired accessToken and a valid refreshToken + clientId
When refreshAccessToken() is called
Then it exchanges the refreshToken for a new accessToken via Spotify's /api/token endpoint
And persists the updated tokens to the config file
```

## AC-4: Spotify HTTP Client GET
```gherkin
Given a valid SpotifyConfig with a non-expired accessToken
When spotifyClient.get("/v1/me/tracks") is called
Then it sends a GET request to https://api.spotify.com/v1/me/tracks with Bearer authorization
And returns the parsed JSON response
```

## AC-5: Spotify HTTP Client POST
```gherkin
Given a valid SpotifyConfig with a non-expired accessToken
When spotifyClient.post("/v1/users/{id}/playlists", body) is called
Then it sends a POST request with JSON body and Bearer authorization
And returns the parsed JSON response
```

## AC-6: Auto-Refresh on Expired Token
```gherkin
Given a SpotifyConfig with an expired accessToken but valid refreshToken
When any client method is called
Then the client automatically refreshes the token before making the API request
And persists the new tokens
```

<task type="auto">
  <name>Task 2: Create Spotify auth and HTTP client with auto-refresh</name>
  <files>src/spotify/auth.ts, src/spotify/spotify-client.ts, test/spotify-auth.test.ts, test/spotify-client.test.ts</files>
  <action>
    **src/spotify/auth.ts:**
    - `refreshAccessToken(config: SpotifyConfig): Promise<{ accessToken: string; expiresAt: number }>` 
      - POST to `SPOTIFY_TOKEN_ENDPOINT` with `grant_type=refresh_token`, `refresh_token`, and `client_id` as `application/x-www-form-urlencoded` body
      - PKCE refresh does NOT require client_secret (public client)
      - Parse response JSON for `access_token` and `expires_in`
      - Compute `expiresAt = Math.floor(Date.now() / 1000) + expires_in`
      - Return new accessToken and expiresAt
      - Throw descriptive error on failure (include status code and response body)
    - `isTokenExpired(config: SpotifyConfig): boolean`
      - Returns true if `expiresAt` is missing or `Date.now() / 1000 >= expiresAt - 60` (60-second buffer)
    - `generatePKCEChallenge(): { codeVerifier: string; codeChallenge: string }`
      - Generate random 43-128 char code_verifier using crypto.randomBytes
      - Compute code_challenge as base64url(sha256(code_verifier))
      - This will be used by the `/spotify auth` command in Phase 11, but define it now
    - `buildAuthURL(clientId: string, redirectUri: string, codeChallenge: string): string`
      - Build Spotify authorization URL with: response_type=code, client_id, scope (from constants), redirect_uri, code_challenge_method=S256, code_challenge
      - Returns the full URL string

    **src/spotify/spotify-client.ts:**
    - `SpotifyClient` class mirroring `AppleMusicClient`:
      - Constructor takes `SpotifyConfig` and a `saveConfig: (config: SpotifyConfig) => void` callback
      - Private `ensureValidToken()`: if `isTokenExpired(config)` and refreshToken exists, call `refreshAccessToken`, update internal config, and call `saveConfig` to persist
      - Private `getHeaders()`: returns `{ Authorization: "Bearer {accessToken}", "Content-Type": "application/json" }`
      - `async get(path: string): Promise<unknown>` — calls ensureValidToken, then fetch GET to `SPOTIFY_API_BASE + path`
      - `async post(path: string, body: unknown): Promise<unknown>` — same pattern for POST
      - Error handling: throw with status, statusText, and response body (match Apple Music client pattern)
    - `createSpotifyClient(config: SpotifyConfig): SpotifyClient` factory function

    **test/spotify-auth.test.ts:**
    - Test `isTokenExpired` returns true when no expiresAt, when expired, when within 60s buffer
    - Test `isTokenExpired` returns false when token has >60s remaining
    - Test `generatePKCEChallenge` produces valid verifier (43-128 chars) and challenge (base64url)
    - Test `buildAuthURL` includes all required params
    - Test `refreshAccessToken` with mocked fetch (mock global.fetch): success case returns new token, error case throws

    **test/spotify-client.test.ts:**
    - Test `get` sends correct headers and returns parsed JSON (mock fetch)
    - Test `post` sends body and correct headers
    - Test auto-refresh: when token expired + refreshToken present, client refreshes before request
    - Test error handling: non-ok response throws descriptive error
