## Purpose

## Output

## Rendering API
- `renderCall(args, theme, context)` → returns `Text` component
  - `theme.fg(color, text)` — apply color: "toolTitle", "muted", "dim", "success", "error", "warning"
  - `theme.bold(text)` — bold text
  - `new Text(content, paddingX, paddingY)` — always use (0, 0) padding
  - `context.lastComponent` — reuse existing Text for streaming updates
- `renderResult(result, { expanded, isPartial }, theme, context)` → returns `Text` component
  - `expanded` — whether user has expanded the result (toggle with keybinding)
  - `isPartial` — streaming/in-progress result
  - `result.content` — array of content parts
  - `result.details` — tool-specific metadata
- Import: `import { Text } from "@mariozechner/pi-tui"`

    1. Create `src/render.ts` with rendering functions for each tool:

       **Library tool:**
       - `renderLibraryCall(args, theme, context)`:
         - Format: `🎵 Library {action}` with action in muted color
         - If limit specified: append ` (limit: {limit})` in dim
       - `renderLibraryResult(result, { expanded, isPartial }, theme, context)`:
         - If isPartial: show spinner "Fetching library data..."
         - Extract first line of text content as summary
         - If result contains "Error": show with error color
         - Default: show `✓` + first line (header like "Library Songs (5 of 150)")
         - If expanded: show full text content

       **Search tool:**
       - `renderSearchCall(args, theme, context)`:
         - Format: `🔍 Search "{term}"` with term in muted
         - If limit: append ` (limit: {limit})` in dim
       - `renderSearchResult(result, { expanded, isPartial }, theme, context)`:
         - If isPartial: "Searching catalog..."
         - Extract match count from header if present
         - Show `✓ {N} results` or `○ No results` or error
         - If expanded: show full results

       **Playlist tool:**
       - `renderPlaylistCall(args, theme, context)`:
         - If action is "create": `📝 Create playlist "{name}"`
         - If action is "add-tracks": `➕ Add tracks to {playlistId}`
         - Append track count in dim
       - `renderPlaylistResult(result, { expanded, isPartial }, theme, context)`:
         - If isPartial: "Creating playlist..." or "Adding tracks..."
         - If success: `✓ Playlist created` or `✓ Tracks added` with details
         - If error: show error
         - If expanded: show full result

       All functions should:
       - Reuse `context.lastComponent` as Text when available (for streaming)
       - Return `new Text(content, 0, 0)`
       - Handle missing/undefined args gracefully
