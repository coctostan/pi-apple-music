  <name>Task 2: Implement Apple Music auth and config system</name>
  <files>src/auth.ts, src/config.ts, src/apple-music-client.ts, test/auth.test.ts</files>
  <action>
    1. Create `src/config.ts`:
       - `loadConfig(): AppleMusicConfig | null` â€” reads from ~/.pi-apple-music/config.json, returns null if not found
       - `saveConfig(config: AppleMusicConfig): void` â€” writes to ~/.pi-apple-music/config.json, creates directory if needed
       - `isConfigured(config: AppleMusicConfig | null): boolean` â€” checks teamId, keyId, privateKeyPath are all present
       - Use `node:fs` and `node:path` for file operations, `node:os` for homedir

    2. Create `src/auth.ts`:
       - `generateDeveloperToken(config: AppleMusicConfig): string`
         - Read the .p8 private key file from config.privateKeyPath
         - Create JWT with header: { alg: "ES256", kid: config.keyId }
         - Claims: { iss: config.teamId, iat: now, exp: now + 15777000 (6 months max) }
         - Sign with `jsonwebtoken` sign() using the private key and ES256 algorithm
         - Return the signed token string
       - Cache the generated token in memory (regenerate only when expired or config changes)

    3. Create `src/apple-music-client.ts`:
       - `AppleMusicClient` class:
         - Constructor takes AppleMusicConfig
         - `private getHeaders(): Record<string, string>` â€” returns Authorization + optional Music-User-Token headers
         - `async get(path: string): Promise<any>` â€” GET request to API_BASE_URL + path with auth headers using `fetch()`
         - `async post(path: string, body: unknown): Promise<any>` â€” POST request with auth headers and JSON body
         - Basic error handling: check response.ok, throw descriptive errors with status codes
       - Export a factory: `createClient(config: AppleMusicConfig): AppleMusicClient`

    4. Create `test/auth.test.ts`:
       - Test that `generateDeveloperToken` produces a valid JWT structure (3 dot-separated base64 segments)
       - Test that the JWT header contains alg: "ES256" and the correct kid
       - Test that the JWT payload contains correct iss, iat, and exp claims
       - Use a test .p8 key (generate an EC P-256 key for testing, NOT a real Apple key)
       - Test that `loadConfig` returns null when no config file exists
       - Test that `saveConfig` + `loadConfig` round-trips correctly (use a temp directory)

    Avoid: Do NOT store credentials in the project directory or commit them â€” always use ~/.pi-apple-music/
    Avoid: Do NOT hardcode any real Apple credentials in tests â€” use generated test keys
