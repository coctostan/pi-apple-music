<objective>
## Goal
Implement the `apple_music_library` tool to fetch real data from the Apple Music API â€” songs, artists, albums, playlists, and recently played â€” and return it as structured, LLM-readable content.

## Purpose
This is the core data layer that enables the LLM to understand the user's music library. Without it, playlist generation would have no context about what the user listens to. The tool output is designed to be directly consumable by the LLM for analysis and recommendation.

## Output
- Working `apple_music_library` tool that fetches real data from Apple Music API
- Library service module with paginated fetching and response formatting
- Type definitions for Apple Music API responses
- LLM-optimized text output (structured markdown, not raw JSON)
</objective>

## API Reference
- Songs: `GET /v1/me/library/songs` â†’ `{ data: [{ id, type, attributes: { name, artistName, albumName, genreNames, durationInMillis, ... } }], next, meta: { total } }`
- Artists: `GET /v1/me/library/artists` â†’ `{ data: [{ id, type, attributes: { name } }], next, meta: { total } }`
- Albums: `GET /v1/me/library/albums` â†’ `{ data: [{ id, type, attributes: { name, artistName, trackCount, genreNames, releaseDate, dateAdded } }], next, meta: { total } }`
- Playlists: `GET /v1/me/library/playlists` â†’ `{ data: [{ id, type, attributes: { name, dateAdded, canEdit, isPublic } }], next, meta: { total } }`
- Recently played: `GET /v1/me/recent/played` â†’ `{ data: [{ id, type, attributes: { name, ... } }] }` (max 10)
- All endpoints require `Authorization: Bearer {dev_token}` + `Music-User-Token: {user_token}` headers
- Pagination via `limit` (query param) and `next` (response field) / `offset`

  <action>
    1. Extend `src/types.ts` with Apple Music API response types:
       - `AppleMusicResource<T>`: `{ id: string, type: string, href?: string, attributes: T }`
       - `AppleMusicResponse<T>`: `{ data: AppleMusicResource<T>[], next?: string, meta?: { total: number } }`
       - `LibrarySongAttributes`: `{ name: string, artistName: string, albumName: string, genreNames: string[], durationInMillis: number, trackNumber?: number }`
       - `LibraryArtistAttributes`: `{ name: string }`
       - `LibraryAlbumAttributes`: `{ name: string, artistName: string, trackCount: number, genreNames: string[], releaseDate?: string, dateAdded?: string }`
       - `LibraryPlaylistAttributes`: `{ name: string, dateAdded?: string, canEdit?: boolean, isPublic?: boolean, description?: { standard?: string } }`
       - `RecentlyPlayedAttributes`: `{ name: string, artistName?: string, artwork?: { url?: string } }`

    2. Create `src/library.ts` â€” the library service:
       - `fetchLibrarySongs(client: AppleMusicClient, limit: number): Promise<string>`
         - GET `/v1/me/library/songs?limit={limit}`
         - Parse response, format each song as: `"{name}" by {artistName} ({albumName}) [{genres}]`
         - Return formatted text with header: `## Library Songs ({count} of {total})\n` + list
       - `fetchLibraryArtists(client: AppleMusicClient, limit: number): Promise<string>`
         - GET `/v1/me/library/artists?limit={limit}`
         - Format: numbered list of artist names
         - Return with header: `## Library Artists ({count} of {total})\n`
       - `fetchLibraryAlbums(client: AppleMusicClient, limit: number): Promise<string>`
         - GET `/v1/me/library/albums?limit={limit}`
         - Format: `"{name}" by {artistName} ({trackCount} tracks) [{genres}]`
         - Return with header: `## Library Albums ({count} of {total})\n`
       - `fetchLibraryPlaylists(client: AppleMusicClient, limit: number): Promise<string>`
         - GET `/v1/me/library/playlists?limit={limit}`
         - Format: playlist name + track count if available
         - Return with header: `## Library Playlists ({count} of {total})\n`
       - `fetchRecentlyPlayed(client: AppleMusicClient): Promise<string>`
         - GET `/v1/me/recent/played?limit=10`
         - Format: `"{name}" ({type})` for each item
         - Return with header: `## Recently Played\n`
       - `fetchLibrarySummary(client: AppleMusicClient): Promise<string>`
         - Make parallel requests: songs (limit=10), artists (limit=10), albums (limit=10), playlists (limit=5)
         - Extract totals from `meta.total` in each response
         - Return a combined summary:
           ```
           ## Library Summary
           - Songs: {total}
           - Artists: {total}
           - Albums: {total}
           - Playlists: {total}

           ### Sample Songs
           {top 10 songs formatted}

           ### Sample Artists
           {top 10 artists}

           ### Sample Albums
           {top 10 albums}
           ```
       - All formatting functions should handle empty responses gracefully ("No {type} found in library")
       - All functions should catch API errors and return descriptive error messages rather than throwing

    Avoid: Do NOT return raw JSON to the LLM â€” always return formatted markdown text
    Avoid: Do NOT fetch all items when a limit is specified â€” respect pagination limits
