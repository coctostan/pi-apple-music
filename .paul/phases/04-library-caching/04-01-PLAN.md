<objective>
## Goal
Add an in-memory cache layer for Apple Music library data with TTL-based expiration, so repeated tool calls don't re-fetch from the API. Support manual cache invalidation via the `/apple-music` command.

## Purpose
Library fetches are slow for large collections and the data rarely changes mid-session. Caching eliminates redundant API calls — especially important when the LLM reads the library summary, then drills into songs, then artists within one conversation.

## Output
- Cache module with TTL-based expiration and per-key storage
- Library functions use cache transparently (fetch on miss, return on hit)
- `/apple-music cache-clear` command to invalidate manually
- `/apple-music status` shows cache state (entries, age)
</objective>

<task type="auto">
  <name>Task 1: Create cache module</name>
  <files>src/cache.ts, src/constants.ts, test/cache.test.ts</files>
  <action>
    1. Add to `src/constants.ts`:
       - `CACHE_TTL_MS = 5 * 60 * 1000` (5 minutes default)

    2. Create `src/cache.ts`:
       - `interface CacheEntry<T>` — `{ data: T, timestamp: number }`
       - `class LibraryCache`:
         - Private `entries: Map<string, CacheEntry<string>>` — keyed by `"action:limit"` string
         - `get(key: string): string | null` — returns cached data if entry exists and not expired, null otherwise
         - `set(key: string, data: string): void` — stores data with current timestamp
         - `clear(): void` — removes all entries
         - `size(): number` — returns number of cached entries
         - `oldestAge(): number | null` — returns age in ms of oldest entry, or null if empty
         - `stats(): { entries: number, oldestAgeSec: number | null }` — returns cache stats
       - Constructor takes optional `ttlMs` parameter (default from CACHE_TTL_MS)
       - Export a singleton: `export const libraryCache = new LibraryCache()`
       - Also export the class for testing with custom TTL

    3. Create `test/cache.test.ts`:
       - Test cache miss returns null
       - Test cache hit returns stored data
       - Test cache expiry (create with 50ms TTL, wait 60ms, verify miss)
       - Test clear removes all entries
       - Test size and stats return correct values
       - Test different keys are independent

    Avoid: Do NOT use file-based or disk caching — in-memory only is sufficient for session-scoped data
  </action>
